# Requirements Document

## Introduction

Апгрейд контента и смыслов агрегатора AI-девушек и ролеплей-сервисов. Цель — превратить сырой каталог в полезный продукт, который реально помогает пользователям выбрать подходящий сервис и конвертирует их в клиентов. Фокус на ценности: понятная навигация, честные сравнения, практические рекомендации по выбору.

Текущее состояние: есть готовый дизайн (IDS), 15 инструментов, 5 коллекций, 4 блог-статьи. Контент сырой, не помогает в выборе.

## Glossary

- **AI-девушка (AI Girlfriend)**: Сервис для общения с виртуальным AI-персонажем в романтическом/ролеплей контексте
- **Ролеплей**: Формат общения с AI, где пользователь и AI играют роли в определённом сценарии
- **Каталог**: Страница со списком всех сервисов с фильтрами
- **Карточка сервиса**: Превью сервиса в каталоге (название, цена, бейджи)
- **Страница сервиса**: Детальная страница с полным описанием, FAQ, тарифами
- **Коллекция**: Тематическая подборка сервисов (например, "Telegram-боты", "Бесплатные")
- **Конверсия**: Переход пользователя на сайт сервиса по партнёрской ссылке
- **VPN**: Инструмент для обхода блокировок, требуется для некоторых сервисов
- **Карты РФ**: Банковские карты российских банков (Сбербанк, Тинькофф и др.)

## Requirements

### Requirement 1: Улучшение главной страницы для понимания ценности сервиса

**User Story:** Как посетитель, я хочу сразу понять что это за сайт и как он мне поможет, чтобы решить стоит ли изучать каталог.

#### Acceptance Criteria

1. WHEN пользователь открывает главную страницу THEN система SHALL отображать заголовок, чётко объясняющий ценность сервиса (помощь в выборе AI-девушки)
2. WHEN пользователь видит hero-секцию THEN система SHALL показывать 3 ключевых преимущества каталога (сравнение, фильтры по доступности из РФ, честные обзоры)
3. WHEN пользователь просматривает главную THEN система SHALL отображать быстрые ссылки на ключевые сценарии выбора (без VPN, с картами РФ, бесплатные)

### Requirement 2: Создание системы рекомендаций по сценариям использования

**User Story:** Как пользователь из России, я хочу быстро найти сервис под мой конкретный сценарий, чтобы не тратить время на изучение неподходящих вариантов.

#### Acceptance Criteria

1. WHEN пользователь ищет сервис без VPN THEN система SHALL показывать отфильтрованный список сервисов, работающих без VPN
2. WHEN пользователь ищет сервис с оплатой картой РФ THEN система SHALL показывать только сервисы, принимающие российские карты
3. WHEN пользователь хочет бесплатный сервис THEN система SHALL показывать сервисы с бесплатным тарифом и указывать реальные лимиты
4. WHEN пользователь выбирает сценарий THEN система SHALL показывать топ-3 рекомендации для этого сценария с кратким объяснением почему

### Requirement 3: Улучшение контента страниц сервисов для помощи в принятии решения

**User Story:** Как пользователь, я хочу получить честную и полную информацию о сервисе, чтобы принять обоснованное решение о регистрации.

#### Acceptance Criteria

1. WHEN пользователь открывает страницу сервиса THEN система SHALL отображать структурированный блок "Для кого подходит" с конкретными сценариями
2. WHEN пользователь читает описание THEN система SHALL показывать честные плюсы и минусы сервиса
3. WHEN сервис требует VPN или не принимает карты РФ THEN система SHALL показывать практические инструкции по обходу ограничений
4. WHEN пользователь просматривает тарифы THEN система SHALL показывать реальную стоимость в рублях (примерный курс) и что входит в каждый план

### Requirement 4: Создание сравнительного контента для выбора между альтернативами

**User Story:** Как пользователь, я хочу сравнить похожие сервисы между собой, чтобы выбрать лучший для моих нужд.

#### Acceptance Criteria

1. WHEN пользователь находится на странице сервиса THEN система SHALL показывать блок "Сравнение с альтернативами" с ключевыми отличиями
2. WHEN пользователь просматривает коллекцию THEN система SHALL отображать сравнительную таблицу всех сервисов в коллекции
3. WHEN пользователь сравнивает сервисы THEN система SHALL показывать критерии: качество ответов, цензура, цена, доступность из РФ

### Requirement 5: Оптимизация коллекций для конкретных потребностей

**User Story:** Как пользователь с конкретной потребностью, я хочу найти подборку сервисов под мой запрос, чтобы сузить выбор до релевантных вариантов.

#### Acceptance Criteria

1. WHEN пользователь открывает коллекцию THEN система SHALL показывать вводный текст, объясняющий для кого эта подборка
2. WHEN пользователь просматривает коллекцию THEN система SHALL отображать рекомендацию "Лучший выбор" с обоснованием
3. WHEN коллекция содержит сервисы THEN система SHALL сортировать их по релевантности для данной категории (не только по рейтингу)

### Requirement 6: Улучшение навигации и поиска для быстрого нахождения нужного

**User Story:** Как пользователь, я хочу быстро найти сервис по моим критериям, чтобы не просматривать весь каталог вручную.

#### Acceptance Criteria

1. WHEN пользователь использует фильтры в каталоге THEN система SHALL показывать количество результатов для каждого фильтра
2. WHEN пользователь применяет несколько фильтров THEN система SHALL комбинировать их логически (AND) и показывать релевантные результаты
3. WHEN результатов по фильтрам нет THEN система SHALL предлагать альтернативные варианты или ослабление фильтров

### Requirement 7: Добавление социального доказательства и актуальности

**User Story:** Как пользователь, я хочу видеть что информация актуальна и проверена, чтобы доверять рекомендациям.

#### Acceptance Criteria

1. WHEN пользователь просматривает сервис THEN система SHALL показывать дату последнего обновления информации
2. WHEN информация о сервисе устарела более 3 месяцев THEN система SHALL показывать предупреждение о возможной неактуальности
3. WHEN сервис имеет особенности (изменились цены, добавили функции) THEN система SHALL отображать бейдж "Обновлено" с кратким описанием изменений

### Requirement 8: Удаление нерелевантного контента

**User Story:** Как владелец сервиса, я хочу убрать контент, который не помогает пользователям выбрать AI-девушку, чтобы сфокусировать сайт на основной ценности.

#### Acceptance Criteria

1. WHEN сервис не относится к AI-девушкам/ролеплею (например, Poe, Stable Diffusion) THEN система SHALL либо удалить его, либо переместить в отдельную категорию "Другие AI-инструменты"
2. WHEN коллекция "image-generators" содержит генераторы изображений без функции общения THEN система SHALL переименовать её в "Генераторы AI-фото" и позиционировать как дополнение к чат-ботам
3. WHEN блог-статья не помогает в выборе сервиса THEN система SHALL либо переписать её с фокусом на практическую пользу, либо удалить
